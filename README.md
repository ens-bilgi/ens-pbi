# ens-pbi

Power BI and Microsoft Fabric development environment for Claude Code. Build semantic models, reports, and dataflows using AI-assisted workflows with modern, source-controllable file formats (PBIR, TMDL, PBIP).

## What This Does

This project lets you use Claude Code (or any AI agent) to:

- **Create semantic models** — define tables, columns, DAX measures, relationships using TMDL (text-based, version-controllable)
- **Build reports** — generate report pages and visuals by writing PBIR JSON files (no drag-and-drop needed)
- **Deploy to Power BI Service** — push content to Fabric workspaces via API or Git sync
- **Query deployed models** — execute DAX queries against live semantic models
- **Manage workspaces** — create and configure Fabric workspaces via REST API

It assembles existing open-source tools from Microsoft and the community rather than reinventing the wheel.

## Prerequisites

| Requirement | Purpose |
|---|---|
| Python 3.10+ | Running deployment scripts and Python tools |
| [uv](https://docs.astral.sh/uv/) | Python package manager (`pip install uv` or `curl -LsSf https://astral.sh/uv/install.sh \| sh`) |
| Node.js 18+ | Running the Fabric MCP Server via `npx` |
| Azure Entra ID app registration | API access to Power BI Service (see [Azure Setup](#azure-setup)) |
| Power BI Pro or Premium Per User license | Required for API operations |
| Claude Code | AI agent that uses the skills and MCP servers in this project |

Optional:
- **Azure CLI** (`az`) — for `az rest` commands used by some skills
- **Power BI Desktop** (Windows only) — for creating initial PBIP project boilerplate
- **Tabular Editor** — for advanced C# scripting and Best Practice Analyzer

## Quick Start

```bash
# 1. Clone the repo
git clone https://github.com/ens-bilgi/ens-pbi.git
cd ens-pbi

# 2. Install Python dependencies
uv sync

# 3. Set up credentials
cp .env.example .env
# Edit .env with your Azure Entra ID credentials (see docs/azure-setup.md)

# 4. Open with Claude Code
claude  # from the project root — CLAUDE.md and .mcp.json are auto-loaded
```

### Where Things Go

- **`projects/`** — PBIP project folders. Each project has a `.SemanticModel/` (TMDL) and `.Report/` (PBIR) subfolder. These are what get deployed to Power BI Service.
- **`skills/`** — Reference material and instruction sets that Claude reads to understand how to write PBIR JSON, TMDL, and interact with Fabric APIs. You generally don't edit these.
- **`scripts/`** — Utility scripts for deployment and automation.

## File Formats

This project uses Power BI's modern text-based formats instead of binary `.pbix` files:

| Format | What It Is | File Type | Location |
|---|---|---|---|
| **PBIP** | Power BI Project — folder-based container | Folder | `projects/<name>.pbip` |
| **PBIR** | Enhanced Report Format — one JSON file per visual/page | JSON | `<name>.Report/definition/pages/` |
| **TMDL** | Tabular Model Definition Language — one file per table | Text (YAML-like) | `<name>.SemanticModel/definition/tables/` |

These formats are Git-friendly: you get meaningful diffs, can do code review on report changes, and merge branches.

## MCP Servers

Two MCP servers are registered in `.mcp.json`:

| Server | What It Does |
|---|---|
| **Fabric MCP** (`@microsoft/fabric-mcp`) | Full Fabric REST API access — workspaces, items, Git integration, job execution. Runs locally via `npx`. |
| **PowerBIQuery** (Remote) | Execute DAX queries against deployed semantic models. Hosted by Microsoft at `api.fabric.microsoft.com`. |

When you open Claude Code in this directory, these servers start automatically and provide tools Claude can call.

## Deployment

Two options for getting PBIP content into Power BI Service:

### Option A: API Deployment (fabric-cicd)

```bash
# Deploy a specific project to a workspace
uv run python scripts/deploy.py \
  --project projects/my-dashboard \
  --workspace-id <workspace-guid>

# Deploy with interactive browser auth (for local dev)
uv run python scripts/deploy.py \
  --project projects/my-dashboard \
  --workspace-name "My Workspace"

# Deploy with service principal (for CI/CD)
uv run python scripts/deploy.py \
  --project projects/my-dashboard \
  --workspace-id <workspace-guid> \
  --spn-auth
```

### Option B: Git Sync

Connect your Fabric workspace to this repo via Fabric Git integration (see Step 8 in `docs/azure-setup.md`). Then any commit/push to the configured branch auto-deploys.

## Azure Setup

Follow `docs/azure-setup.md` for the full walkthrough. Summary:

1. Register an app in Microsoft Entra ID
2. Create a client secret
3. Add Power BI API permissions and grant admin consent
4. Create a security group containing the app
5. Enable service principal access in Power BI admin portal
6. Add the service principal to your target workspace(s)
7. Copy credentials to `.env`
8. (Optional) Set up Fabric Git integration

This requires coordination with your Azure/Power BI admin.

## Typical Workflow

1. **Create initial PBIP boilerplate** — Use Power BI Desktop (Windows) or Power BI web authoring (app.powerbi.com) to create a blank report connected to your data source. Save as PBIP format. Place in `projects/`.

2. **Ask Claude to build it out** — e.g., "Add a KPI dashboard page with revenue cards and a trend chart to the sales report."

3. **Claude writes TMDL + PBIR** — adds measures to `.SemanticModel/definition/tables/`, creates page/visual JSON in `.Report/definition/pages/`.

4. **Deploy** — run `deploy.py` or push to trigger Git sync.

5. **Iterate** — open the report in Power BI Service, review, ask Claude for changes.

## macOS Note

Power BI Desktop is Windows-only. On macOS, create initial PBIP projects via:
- Power BI web authoring at app.powerbi.com
- Fabric REST API (the `powerbi-authoring-cli` skill can do this)
- A Windows VM or colleague's machine

Once the initial PBIP boilerplate exists, all subsequent editing happens in text files that Claude can write on any OS.

## Troubleshooting

### MCP server won't start
- Check Node.js is installed: `node --version` (need 18+)
- Try running manually: `npx -y @microsoft/fabric-mcp@latest server start --mode all`

### Authentication errors (401/403)
- Verify `.env` credentials match your Entra ID app registration
- Check admin consent was granted (Step 3 in azure-setup.md)
- Confirm service principal is in the workspace with Member/Admin role

### Deployment fails
- Ensure `POWERBI_WORKSPACE_ID` is set or pass `--workspace-id`
- Check that the PBIP project has valid `.SemanticModel/` and `.Report/` folders
- Try `uv run python scripts/deploy.py --help` for all options

### Claude doesn't see Power BI tools
- Restart Claude Code from the project root directory
- Check `.mcp.json` is present and valid JSON
- Run `claude mcp list` to see registered servers
